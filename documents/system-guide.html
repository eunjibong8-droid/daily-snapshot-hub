Daily Snapshot Hub

GitHub Actions 기반 일일 데이터 수집·아카이빙 허브. 여러 수집 모듈이 독립적으로 동작하며, 수집 결과를 매일 repo에 커밋해 쌓아둡니다. 하나의 모듈이 실패해도 나머지는 계속 동작합니다.

📰 RSS 다이제스트

8개 피드에서 새 글을 매일 수집해 Markdown으로 아카이빙

🌤️ 날씨·미세먼지

서울·부산 기온·강수·PM10·PM2.5를 CSV로 월별 누적

📊 주간 리포트

매주 월요일, 지난 주 RSS + 날씨를 한 페이지로 통합

💡
Public repo로 운영됩니다. AWS 키·개인 API 키는 절대 커밋하지 않고 secrets. 네임스페이스만 사용합니다. 수집 데이터도 공개돼도 무방한 것만 포함합니다.

중복 제거 방식

state/rss-last-seen.json에 피드별 마지막 수집 시각(UTC ISO)을 저장합니다. 다음 실행 시 해당 시각 이후에 발행된 항목만 수집합니다. 첫 실행에는 전체 항목을 수집합니다.

📊 모듈 3

주간 통합 리포트

매주 월요일 KST 08:30, 직전 ISO 주차(월~일)의 RSS + 날씨 데이터를 읽어 한 페이지로 통합합니다. 크로스 모듈 집계가 이 프로젝트의 핵심 가치입니다.

리포트 포함 내용

섹션내용
🌤️ 날씨 요약위치별 주간 평균·최고·최저 기온, 총 강수량, PM10·PM2.5 평균, 미세먼지 등급
📰 RSS 하이라이트이번 주 총 수집 건수, 피드별 건수 표
피드별 상위 글각 피드에서 최대 5개 제목 + 링크
⚙️ GitHub Actions

워크플로우 구조

각 모듈은 독립된 워크플로우 파일로 관리됩니다. concurrency: group: commit-main으로 동시 커밋 충돌을 방지합니다.

공통 패턴

jobs:
  collect-and-commit:
    runs-on: ubuntu-latest
    permissions:
      contents: write        # git push 권한

    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v5
      - run: uv sync --frozen
      - run: uv run python -m daily_hub.collectors.rss   # 모듈별 변경
      - name: Commit and push
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add snapshots/rss state/rss-last-seen.json
          git diff --staged --quiet || git commit -m "chore(rss): daily snapshot $(date +%Y-%m-%d)"
          git push
⚠️
수동 트리거 테스트는 GitHub Actions 탭 → 워크플로우 선택 → Run workflow 버튼으로 언제든 가능합니다.

🔧 설정 가이드

설정 파일 가이드

RSS 피드 추가 · 제거

config/rss-feeds.yml에서 피드를 추가하거나 제거하면 코드 변경 없이 바로 적용됩니다.

feeds:
  - name: My New Feed
    url: https://example.com/feed
    category: 카테고리명

날씨 수집 도시 추가

config/locations.yml에 좌표를 추가합니다. 위도·경도는 Google Maps에서 확인할 수 있습니다.

locations:
  - name: Daegu
    name_ko: 대구
    latitude: 35.8714
    longitude: 128.6014
    timezone: Asia/Seoul

uv run python -m daily_hub.collectors.weather

# 주간 리포트 수동 생성
uv run python -m daily_hub.reporters.weekly
